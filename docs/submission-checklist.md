# Beatrun Submission Checklist

Use this checklist for the Qidi-track initial submission and later competition rounds.

## Identity

- [ ] Final project name: Beatrun
- [ ] Final team name
- [ ] Team members and roles
- [ ] Advisor name and school information
- [ ] Consistent Chinese project title
- [ ] Consistent English project title if needed

## Initial Submission

- [ ] Competition registration completed on the official site
- [ ] Official work description document filled with the required template
- [ ] Current app build verified
- [ ] App screenshots exported
- [ ] Optional poster or effect image prepared
- [ ] README updated to match the submitted app state
- [ ] Audio copyright statement prepared
- [ ] Final package backed up before 2026-06-30 23:59

## Work Description Document

- [ ] Problem background: cadence training and rhythm stability
- [ ] Target users: beginner runners, cadence-training runners, school running groups
- [ ] Core workflow: cadence selection, legal demo music matching, 1:1 tempo adjustment, synced playback
- [ ] Innovation points
- [ ] Technical architecture
- [ ] Current completed functions
- [ ] Demo limitations and future work
- [ ] Market or social value
- [ ] Copyright and data compliance statement

## Demo Video

- [ ] 2-3 minute script written
- [ ] Fresh-launch demo recorded
- [ ] Cadence change demo recorded
- [ ] Music recommendation and match score explained
- [ ] Original BPM, adjusted BPM, speed change, and rights status shown
- [ ] Sync offset and confidence shown
- [ ] Copyright-safe audio explanation included
- [ ] Final video exported and checked on another device

## App Readiness

- [ ] Clean build succeeds
- [ ] Fresh launch selects a recommended track
- [ ] Play/pause works repeatedly
- [ ] Cadence slider refreshes recommendations
- [ ] Vocal/instrumental switch refreshes recommendations
- [ ] Recommendations only include +/-10% 1:1 tempo-adjusted matches
- [ ] No double-time or half-time matching appears in the demo path
- [ ] Metronome volume control works
- [ ] Backing loop volume control works
- [ ] Sync status updates while playing
- [ ] No placeholder text visible in the main demo path
- [ ] No crash during a 3-minute continuous demo

## Evidence To Collect

- [ ] Build command and result
- [ ] Simulator or real-device model
- [ ] Demo screen recording
- [ ] Three to five screenshots
- [ ] Demo catalog rights table
- [ ] Short product architecture diagram
- [ ] Short competitor comparison
- [ ] Any user interview or survey notes

## Final Review

- [ ] The first 30 seconds explain the value clearly
- [ ] The materials do not overclaim unimplemented Apple Watch or music-provider features
- [ ] All screenshots match the current app
- [ ] The video, document, and README use the same feature names
- [ ] Submission package opens correctly after compression
